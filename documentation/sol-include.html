Solid Web Components > sol-include

The sol-include component supports embedding a variety of sources into an HTML page. In some cases sources may be queried using parameters in the sol-include component. For example the queryParam attribute can be used to specifiy replaceable parameters in an included sol-sparql componnet and the wanted attribute can be used to specify CSS and RDFa query selectors.

including other web components

Any web component can be included into an HTML page. It will be inserted into the page's DOM and will therefore be activated as if it had been hard-coded into the page. For example, this includes a previously saved sol-sparql component.

<sol-include
  type="htmlComponent"
  source="assets/test-query.html"
></sol-include>

See also passing a queryParam to an included sol-sparql component.

including an HTML snippet

<sol-include
    type="html"
    source="assets/test-include.html"
></sol-include>

including an HTML snippet as text

You can specify type="text" to display the source code of an HTML snippet. Note that this is the code for the button in the example above and that clicking the button does not produce an alert. If we had omitted the DOMPurify import, the button would be clickable.

<sol-include
    type="text"
    source="assets/test-include.html"
></sol-include>

including a Markdown snippet as HTML

<sol-include
    type="markdown"
    source="assets/test-include.md"
></sol-include>

querying HTML and HTML/RDFa pages

The wanted attribute allows you to specify a CSS/RDFa query selector. For example this component uses the specified wanted attribute to get a list of anchors to the Solid Specifications by querying a web page with a combined CSS/RDFa query selector. 

<sol-include
    type="html"
    source="https://solidproject.org/TR/"
    wanted="#work-item-technical-reports a[rel='cito:citesForInformation']"
></sol-include>

including links from an RSS/Atom feed

<sol-defaults proxy="http://localhost:8080/proxy/?uri="></sol-defaults>
<sol-include
    type="rss"
    source="https://forum.solidproject.org/posts.rss"
></sol-include>

sanitizing included HTML & Markdown snippets

When including HTML and Markdown snippets, as protection against XSS attacks, *sol-include* will automatically call DOMPurify.sanitize before displaying the included snippets. This sanitizing is skipped in the case of including web-components rather than plain HTML/Markdown documents. This means that when ever you use a web-component, you should trust its source just as you would need to trust the source of an imported javascript file in your toolchain.
