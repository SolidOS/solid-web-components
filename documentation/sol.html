Solid Web Components

These components provide access to Solid and other linked-data resources allowing developers and hobbyists to make use of the SolidOS software stack using plain HTML. This is a Work in Progress (WiP) so feedback is welcome using github issues or chatting with me (@jeff-zucker) on the solidos gitter chat. I'll be adding components regularly for a while so if you're interested, you may want to watch the repo.

How to use the components

In order to embed a component in a web page, the page needs two script tags as shown below.

<script src="https://solidos.github.io/solid-ui/dist/solid-ui.min.js" type="module"></script>
<script src="https://solidos.github.io/solid-web-components/sol-core.js" type="module"></script>

Once those tags are on the page simply insert the HTML for the component as shown in the documention and everthing should work. No coding is needed, only the HTML component.

Currently available components

  • sol-sparql - save and embed SPARQL queries in web pages
  • sol-include - embed components and query data from RDFa, HTML, RSS/Atom, and Markdown resources

Note on URLs

Resources may be identified by absolute URLs (https://foo.html), by URLs relative to the current host (/foo.html), or by URLs relative to the current page (foo.html). For purposes of RDF, these are all converted to absolute URLs.

For Advanced Users : Building the libraries

Instead of using CDNs of the libraries in the script tags as shown above, you may want to use local versions instead. You can get and build solid-ui from npm and can get (no build needed) sol-core by cloning the github repo.
