sol-query — Script API

Use runQuery to fetch and query RDF data directly from JavaScript — no web component needed. Returns plain JS values ready to use in your code.

Setup

Load rdflib first

rdflib must be available before calling runQuery. The simplest approach uses an importmap so both your script and rdf-utils.js share the same rdflib instance:

In your HTML <head>
<script type="importmap">
  {"imports": {"rdflib": "https://esm.sh/rdflib"}}
</script>
In your module script
import { runQuery } from './sol-query/shared/rdf-utils.js';

const rows = await runQuery({ endpoint: 'data.ttl', wanted: '? foaf:name ?' });
The importmap tells the browser to resolve 'rdflib' to the CDN URL for both your script and the sol-query module. No explicit window.$rdf assignment is needed.

Alternative: explicit window.$rdf

If you load rdflib separately (e.g., via a UMD bundle or a different CDN URL), assign it to window.$rdf before calling runQuery:

<!-- UMD bundle sets window.$rdf automatically -->
<script src="https://cdn.jsdelivr.net/npm/rdflib/dist/rdflib.min.js"></script>

<!-- Or assign explicitly from a module -->
<script type="module">
  import * as $rdf from 'https://esm.sh/rdflib';
  window.$rdf = $rdf;
</script>

API

runQuery(options)

import { runQuery } from './sol-query/shared/rdf-utils.js'; const result = await runQuery(options);
OptionTypeDescription
endpoint string required URL of the RDF document (for wanted or default) or SPARQL endpoint (for sparql)
sparql string one of Inline SPARQL text (SELECT …) or URL of an RDF file containing a stored query. Either sparql or wanted is required.
wanted string one of Mini-query pattern — three or more space-separated tokens. endpoint must be an RDF document. See syntax below.
vars string[] optional Restrict which columns appear in the output. Default: all columns returned by the query.

SolQuery.run(options)

Identical to runQuery; available as a static method on the custom element class:

import { SolQuery } from './sol-query/sol-query.js'; const result = await SolQuery.run(options);

Return values

runQuery returns a Promise that resolves to one of:

ConditionReturns
1 row × 1 column string | null — the value directly, not wrapped in an array
Multiple rows or columns Array<Object> — one object per row; keys are column names, values are strings
No results [] — empty array

Column names for mini-queries and default queries are s, p, o (whichever positions were wildcards). For SPARQL queries the column names match the variable names in the SELECT clause.

Examples
// 1 row × 1 col → scalar
const name = await runQuery({ endpoint, sparql: 'SELECT ?n WHERE { <X> foaf:name ?n }' });
// name === "Tim Berners-Lee"

// Multiple rows → array
const friends = await runQuery({ endpoint, wanted: '<X> foaf:knows ?' });
// friends === [{ o: 'http://...' }, { o: 'http://...' }, ...]

// Filter columns with vars
const labels = await runQuery({ endpoint, sparql: 'SELECT ?s ?p ?o WHERE {…}', vars: ['s'] });
// labels === [{ s: 'http://...' }, ...]

Mini-query syntax

A mini-query is a space-separated triple pattern passed as the wanted option. The endpoint is fetched as an RDF document and matched against the pattern using rdflib's store.

TokenMeaningExample
?Wildcard — matches anything?
?nameAlso a wildcard (name is ignored)?subject
<uri>Full IRI in angle brackets<http://xmlns.com/foaf/0.1/name>
prefix:localCURIE expanded via built-in prefix tablefoaf:name, schema:Person
"literal"String literal (with optional ^^type or @lang)"Alice", "2024"^^<xsd:date>
multi word textEverything after the predicate that is not an IRI or CURIE becomes a literal? foaf:name john w. smith → searches for literal john w. smith

Built-in prefixes

rdf, rdfs, owl, xsd, foaf, schema, dc, dct, skos, pim, acl, ldp, solid, vcard, sioc, cert, geo, wot, bio, dbo, dbr, dbp

Add extra prefixes by passing them to the lower-level parseWantedParts function.

The returned columns are the wildcard positions only. '? foaf:name ?' returns columns s and o; '<X> ? ?' returns p and o.

Examples

All triples matching a pattern

const rows = await runQuery({
  endpoint: 'https://dbpedia.org/data/Tim_Berners-Lee.ttl',
  wanted: '? ? ?',   // all triples — returns { s, p, o }
});
console.log(rows.length, 'triples');

All predicates and values for a subject

const props = await runQuery({
  endpoint: 'https://dbpedia.org/data/Tim_Berners-Lee.ttl',
  wanted: '<http://dbpedia.org/resource/Tim_Berners-Lee> ? ?',
});
// props === [{ p: 'http://...', o: 'http://...' }, ...]

Find by literal value (multi-word)

const matches = await runQuery({
  endpoint: 'people.ttl',
  wanted: '? foaf:name john w. smith',
  // treated as: null, foaf:name node, literal("john w. smith")
});
// matches === [{ s: 'http://...' }]

SPARQL — scalar result

const birthDate = await runQuery({
  endpoint: 'https://dbpedia.org/sparql',
  sparql: `SELECT ?d WHERE {
    <http://dbpedia.org/resource/Tim_Berners-Lee>
    <http://dbpedia.org/ontology/birthDate> ?d
  } LIMIT 1`,
  vars: ['d'],
});
// birthDate === "1955-06-08" (string, not array)

SPARQL — multiple rows, filtered columns

const names = await runQuery({
  endpoint: 'https://dbpedia.org/sparql',
  sparql: `SELECT ?name ?place WHERE {
    ?p a <http://dbpedia.org/ontology/Person> ;
       foaf:name ?name ;
       dbo:birthPlace ?place .
    FILTER(lang(?name) = 'en')
  } LIMIT 10`,
  vars: ['name'],   // drop ?place from output
});
// names === [{ name: 'Alice' }, { name: 'Bob' }, ...]

Stored SPARQL query

// The RDF file at myQueries.ttl contains a sparql:query triple
const rows = await runQuery({
  endpoint: 'https://example.org/data.ttl',
  sparql: 'https://example.org/myQueries.ttl#FindPeople',
});

Using SolQuery.run() from the component module

import { SolQuery } from './sol-query/sol-query.js';

const row = await SolQuery.run({
  endpoint: 'profile.ttl',
  wanted: '? foaf:name ?',
});
// Same behaviour as runQuery
