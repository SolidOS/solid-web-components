sol-menu Component

A sidebar-nav + content-panel element. Same declarative API as <sol-tabs>: fill with <a href="…">Label</a> children; each anchor becomes a menu entry whose href is loaded into the content pane. Nest <submenu> elements for collapsible submenus of any depth.

Full API reference: SolMenu API docs (generated from source JSDoc)

Attributes

AttributeDescription
orientationDefault is sidebar (vertical nav). Set horizontal to lay the nav on top.
handlerDefault sol-* component tag for anchors that don't set their own handler.

Per-anchor: handler="sol-*" selects which component renders the href. Any additional anchor attributes (wanted, sparql, view, …) are forwarded to the generated element. The href is passed as both source and endpoint.

Event: sol-menu-change fires on navigation, detail: { name }.

Standalone demo

<sol-menu>
  <a href="../data/sample.md">Readme</a>
  <a href="../data/sample-data.ttl" raw>Turtle source</a>
  <a href="../data/sample-data.ttl" handler="sol-query" wanted="? ? ?">Triples table</a>
</sol-menu>

Submenus (collapsible groups)

Nest a <submenu> inside <sol-menu> to make a group. Its <label> is the group heading; anchors (and further <submenu>s) inside become its items. Any depth is supported. Add the open attribute to expand a group by default.

<sol-menu>
  <a href="../data/sample.md">Home</a>
  <submenu open>
    <label>Data</label>
    <a href="../data/sample-data.ttl" raw>Turtle</a>
    <a href="../data/sample-data.ttl" handler="sol-query" wanted="? ? ?">Table</a>
    <submenu>
      <label>Forms</label>
      <a href="../data/contact-form.ttl" raw>Contact form</a>
      <a href="../data/review-form.ttl" raw>Review form</a>
    </submenu>
  </submenu>
</sol-menu>
Home Turtle Table Contact form Review form

As a sol-query view

<sol-query view="menu"> turns a result set into a menu. Uses ?link + ?label vars when present, otherwise the first two columns. A handler on the host is forwarded to the generated <sol-menu>.

<sol-query
  view="menu"
  handler="sol-include"
  endpoint="../data/sample-data.ttl"
  sparql="
    PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
    SELECT ?link ?label WHERE {
      ?link a ?t ; rdfs:label ?label .
    } LIMIT 6
  "></sol-query>
