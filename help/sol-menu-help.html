sol-menu Component

A sidebar-nav + content-panel element. Same declarative API as <sol-tabs>: fill with <a href="…">Label</a> children; each anchor becomes a menu entry whose href is loaded into the content pane. Nest <submenu> elements for collapsible submenus of any depth.

Full API reference: SolMenu API docs (generated from source JSDoc)

Attributes

AttributeDescription
orientationDefault is sidebar (vertical nav). Set horizontal to lay the nav on top.
handlerDefault sol-* component tag for anchors that don't set their own handler.
from-rdfURI (with optional fragment) pointing to a ui:Menu resource. Loads the entire menu structure from RDF. See From RDF below.

Per-anchor: handler="sol-*" selects which component renders the href. Any additional anchor attributes (pattern, sparql, view, …) are forwarded to the generated element. The href is passed as both source and endpoint.

Event: sol-menu-change fires on navigation, detail: { name }.

Styling with CSS custom properties

<sol-menu> uses shadow DOM, so its internals cannot be styled directly. Set these CSS custom properties on the host element or an ancestor to customise appearance.

PropertyDefaultWhat it controls
--font-uisystem sansFont family for the entire component
--font-size16pxNav button font size
--textblackNav button text colour
--border#e0e0e0Nav/content divider and popup border
--hover#f0f0f0Button background on hover
--accent-dark#1976d2Hover and active button text colour
--focus-bg#e3f2fdActive button background
--surface#fffSubmenu popup background

Example:

<sol-menu style="--accent-dark:#e65100; --focus-bg:#fff3e0; --border:#ccc;">
  <a href="page1.html">Page 1</a>
  <a href="page2.html">Page 2</a>
</sol-menu>
The component includes styles/root.css defaults for all these properties. Loading root.css on your page gives you a consistent baseline; override individual properties on the element or a wrapper to customise.

JavaScript API

MemberTypeDescription
itemsget/setArray of { name, render(body), children?, icon? } objects. Setting replaces the nav.
activeItemgetName of the currently selected item (or null).
bodygetThe content-pane DOM element (shadow .sol-menu-content).
select(name)methodActivate a menu item by name (case-insensitive match).

Imperative example:

const m = document.createElement('sol-menu');
m.items = [
  { name: 'Overview', render(body) { body.textContent = 'Hello!'; } },
  { name: 'Details',  render(body) { body.innerHTML = '<p>Detail content</p>'; } },
];
document.body.appendChild(m);
m.select('Overview');

Submenus (collapsible groups)

Nest a <submenu> inside <sol-menu> to make a group. Its <label> is the group heading; anchors (and further <submenu>s) inside become its items. Any depth is supported. Add the open attribute to expand a group by default.

<sol-menu>
  <a href="../data/sample.md">Home</a>
  <submenu open>
    <label>Data</label>
    <a href="../data/sample-data.ttl" raw>Turtle</a>
    <a href="../data/sample-data.ttl" handler="sol-query" pattern="? ? ?">Table</a>
    <submenu>
      <label>Forms</label>
      <a href="../data/contact-form.ttl" raw>Contact form</a>
      <a href="../data/review-form.ttl" raw>Review form</a>
    </submenu>
  </submenu>
</sol-menu>
Home Turtle Table Contact form Review form

From RDF

Use the from-rdf attribute to load the entire menu structure from an RDF document using the ui: vocabulary. The value is a URI with a fragment pointing to the root ui:Menu subject.

<sol-menu from-rdf="menu.ttl#MainMenu"></sol-menu>

A menu's ui:parts list can contain three item types:

TypePurpose
ui:MenuA submenu. Has ui:label, ui:parts (ordered list), and optionally ui:orientation and ui:linkTarget.
ui:LinkA simple link. Has ui:label and either ui:href (URL to load via sol-include or a ui:handler) or ui:contents (inline HTML literal). Optional ui:icon.
ui:ComponentA self-contained web component. Has ui:label (display name), ui:name (element tag, e.g. "sol-query"), and ui:attribute (schema:PropertyValue pairs set as element attributes). Optional ui:icon.
PredicateDescription
ui:linkTargetOn the root ui:Menu, a CSS selector. Content renders into that element instead of the built-in panel.
ui:iconOn a ui:Link or ui:Component, a URL (including data: SVG) displayed in place of the label text.
ui:handlerOn a ui:Link, points to a ui:Component node used to render the href. Prefer using ui:Component directly in ui:parts instead.

See the from-rdf demo page for a full working example with an RDF menu definition and SHACL shape.

As a sol-query view

<sol-query view="menu"> turns a result set into a menu. Uses ?link + ?label vars when present, otherwise the first two columns. A handler on the host is forwarded to the generated <sol-menu>.

<sol-query
  view="menu"
  handler="sol-include"
  endpoint="../data/sample-data.ttl"
  sparql="
    PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
    SELECT ?link ?label WHERE {
      ?link a ?t ; rdfs:label ?label .
    } LIMIT 6
  "></sol-query>
