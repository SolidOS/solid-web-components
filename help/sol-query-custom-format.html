sol-query — Custom View Renderers

When the view attribute is a URL, sol-query dynamically imports that module and calls its render(container, data) export instead of using the built-in table/dl/list renderers.

Module contract

The module at the URL must be a valid ES module that exports a render function:

// my-renderer.js export function render(container, data) { // container — the DOM element inside the sol-query shadow root // data — { vars: string[], results: Array<Row> } container.appendChild( /* your DOM nodes */ ); }

The function receives:

  • container — the cleared <div class="container"> inside the shadow DOM. Append DOM nodes to it directly; do not use innerHTML with untrusted data.
  • data — the raw query result object (see Data shape below).
The module is loaded with a dynamic import(url) call, so it must be served with CORS headers that allow the page's origin. Same-origin or Access-Control-Allow-Origin: * both work.

Usage

<sol-query
  endpoint="data.ttl"
  view="./my-renderer.js"
></sol-query>

<!-- Any URL works — absolute or relative -->
<sol-query
  endpoint="https://dbpedia.org/sparql"
  sparql="SELECT ?name ?birth WHERE { … }"
  view="https://example.org/renderers/timeline.js"
></sol-query>

A view value is treated as a URL (and triggers the custom renderer) when it starts with http://, https://, ./, ../, or /. The built-in values table, dl, and list are never treated as URLs.

Data shape

The data argument has this shape:

{
  vars:    string[],          // column names, e.g. ['name', 'age'] or ['p', 'o']
  results: Array<{            // one object per result row
    [colName]: {
      type:  'uri' | 'literal' | 'bnode',
      value: string,          // the string value
      // uri cells also have data-uri set for dereference clicks
      // bnode cells also have _data: { vars, results } for the node's properties
    }
  }>
}

Column names for mini-queries and default document queries are s, p, o (only the wildcard positions). For SPARQL queries they match the SELECT variable names.

Accessing values in a renderer
export function render(container, { vars, results }) {
  for (const row of results) {
    for (const col of vars) {
      const cell = row[col];
      if (!cell) continue;
      if (cell.type === 'uri') {
        // cell.value is the full URI string
      } else {
        // cell.value is the literal string
      }
    }
  }
}

Example renderer

This renderer (served at .../data/card-renderer.js) displays each result row as a card:

export function render(container, data) {
  const { vars, results } = data;

  const shorten = v => v?.value
    ? v.value.replace(/.*[/#]([^/]+)\/?$/, '$1') || v.value
    : '';

  const grid = document.createElement('div');
  grid.style.cssText = 'display:flex;flex-wrap:wrap;gap:.75rem;padding:.5rem 0';

  results.forEach(row => {
    const card = document.createElement('div');
    card.style.cssText = 'border:1px solid #ddd;border-radius:6px;padding:.75rem 1rem;…';

    vars.forEach(v => {
      const cell = row[v];
      if (!cell?.value) return;
      const line = document.createElement('div');
      // label
      const label = document.createElement('span');
      label.textContent = v + ':';
      line.appendChild(label);
      // value
      if (cell.type === 'uri') {
        const a = document.createElement('a');
        a.href = cell.value;
        a.textContent = shorten(cell);
        line.appendChild(a);
      } else {
        line.appendChild(document.createTextNode(cell.value));
      }
      card.appendChild(line);
    });

    grid.appendChild(card);
  });

  container.appendChild(grid);
}

Live demo

The query below fetches names, birth dates, alma maters, and nationalities for Tim Berners-Lee and Ada Lovelace from DBpedia, rendered as one card per person using the card renderer above:

<sol-query
  endpoint="https://dbpedia.org/sparql"
  sparql="SELECT ?name ?birthDate ?almaMater ?nationality WHERE {
    VALUES ?person {
      <http://dbpedia.org/resource/Tim_Berners-Lee>
      <http://dbpedia.org/resource/Ada_Lovelace>
    }
    ?person foaf:name ?name .
    OPTIONAL { ?person dbo:birthDate ?birthDate }
    OPTIONAL { ?person dbo:almaMater ?almaMater }
    OPTIONAL { ?person dbo:nationality ?nationality }
    FILTER(!isLiteral(?name) || lang(?name) = 'en' || lang(?name) = '')
  } LIMIT 20"
  view="../data/card-renderer.js"
></sol-query>
