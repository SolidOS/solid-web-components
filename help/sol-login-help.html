sol-login : Solid OIDC authentication

  • Drop-in web component for Solid pod authentication
  • Shows a Log in / Log out button with issuer dropdown
  • Manages OIDC sessions via @inrupt/solid-client-authn-browser
  • Provides authenticated fetch for any URL
  • Dispatches sol-login and sol-logout events
  • Auto-propagates auth to sol-query and sol-include on the same page — no wiring needed
  • Patches rdflib's Fetcher and threads the authenticated fetch into Comunica SPARQL queries

Full API reference: SolLogin API docs (generated from source JSDoc)

Loading the auth library: sol-login expects @inrupt/solid-client-authn-browser to be available at window.solidClientAuthn. The simplest setup is two script tags — the prebuilt UMD ships in dist/vendor/: 
<script src="../dist/vendor/@inrupt-solid-client-authn-browser.umd.js"></script>
<script src="../dist/solid-web-components.bundle.min.js"></script>
See install-modes.html for importmap and bundler alternatives.

Usage

1. Basic Usage

Drop a <sol-login> element into your page. It auto-initializes on DOMContentLoaded.

<sol-login></sol-login>

2. Pre-configured Issuers

The issuers attribute provides quick-select buttons in the login dropdown.

<sol-login issuers="https://solidcommunity.net,https://login.inrupt.com"></sol-login>

3. Paired with sol-pod

Point a <sol-pod> at the login element for authenticated browsing.

<sol-login id="auth" issuers="https://solidcommunity.net"></sol-login>
<sol-pod source="https://pod.example/" login="#auth"></sol-pod>

4. With sol-query / sol-include — auto-propagation

Drop a <sol-login> on the page and the authenticated fetch flows to every <sol-query> and <sol-include> automatically — including Comunica SPARQL queries against authenticated endpoints. No login attribute required.

<sol-login issuers="https://solidcommunity.net"></sol-login>

<!-- Authenticated SPARQL via Comunica: -->
<sol-query endpoint="https://your.pod/sparql"
           sparql="SELECT ?s WHERE { ?s a foaf:Person } LIMIT 20"></sol-query>

<!-- Authenticated document include: -->
<sol-include source="https://your.pod/private/notes.md"></sol-include>

If the page has multiple <sol-login> elements (advanced — e.g. one per pod), target a specific one with the login attribute (CSS selector):

<sol-login id="primary"></sol-login>
<sol-login id="alt"></sol-login>
<sol-query login="#alt" endpoint="https://other.pod/sparql" ...></sol-query>

The same login attribute is honored by sol-pod / sol-pod-ops / sol-wac, which still require it (no auto-discovery there yet).

5. Event Handling

Listen for login/logout to trigger app-specific actions.

<sol-login id="auth"></sol-login>
<script>
  document.getElementById('auth').addEventListener('sol-login', (e) => {
    console.log('Logged in as', e.detail.webId);
    console.log('Issuer:', e.detail.issuer);
  });
  document.getElementById('auth').addEventListener('sol-logout', () => {
    console.log('Logged out');
  });
</script>
