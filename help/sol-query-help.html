sol-query : flexibly query Linked Data

  • its a plug-and-play web component - no coding needed
  • supports querying with simple triple patterns or full SPARQL with replaceable parameters
  • presents results as HTML table or HTML description list or in a user-supplied custom template
  • uses the Comunica SPARQL engine with a fallback to the rdflib SPARQL engine
  • handles SPARQL endpoints and simple RDF documents, including RDFa documents
  • uses authenticated Solid fetch when available
  • has safeguards against JavaScript and SPARQL injection
retrieve         using attributes
---------------------------------
all statements : endpoint="RDF DOCUMENT URI"
one subject    : endpoint="SUBJECT URI INSIDE A RDF DOCUMENT"
mini query     : endpoint="RDF DOCUMENT URI" wanted="TRIPLE PATTERN"
RDFa query     : endpoint="RDFa DOCUMENT URI" wanted="CSS SELECTOR PATTERN"
inline SPARQL  : endpoint="SPARQL ENDPOINT" sparql="LITERAL SPARQL STRING"
stored SPARQL  : endpoint="SPARQL ENDPOINT" sparql="URI OF A SPARQL QUERY"
replace param  : endpoint="SPARQL ENDPOINT" sparql="URI OF A SPARQL QUERY" var-paramName="paramValue"

format           using attributes
---------------------------------
as HTML table            : format="table" # or omit format attribute - table is the default
as HTML description list : format="dl"

Usage Details

1. View All Statements in a Document

The endpoint attribute points to an RDF document

<sol-query endpoint="http://example.com/foo.ttl"></sol-query>

2. View All Statements About a Subject

The endpoint attribute points to a subject inside an RDF document

<sol-query endpoint="http://example.com/foo.ttl#BarTopic"></sol-query>

3. Mini-Query Pattern

The endpoint attribute points to an RDF document

The wanted attribute holds a simplified triple query (see below)

<sol-query
  endpoint="http://example.com/foo.ttl"
  wanted="? skos:broader Software"
></sol-query>

Mini-query syntax: Three space-separated parts representing subject, predicate, and object.

  • ? or ?var — wildcard (matches anything)
  • prefix:term — CURIE expanded via built-in prefix table (rdf, rdfs, foaf, schema, dct, skos, ldp, solid…)
  • <http://...> — full URI in angle brackets
  • literal except for URIs and CURIES, all text after the predicate is treated as a single literal.

4. Stored SPARQL Query

The endpoint attribute points to a SPARQL endpoint.

The sparql attribute points to a SPARQL query inside an RDF document.

<sol-query
  endpoint="http://example.com/foo.ttl"
  sparql="http://example.com/MyQueries.ttl#Query1"
></sol-query>

Query storage: The RDF document should contain the query using this format

<#MyQuery>
   a <http://www.w3.org/ns/sparql#query> ;
   <http://www.w3.org/2000/01/rdf-schema#comment> """
     PREFIX schema: <http://schema.org/>
     SELECT ?name WHERE {
       ?person a schema:Person; schema:name ?name.
     }
   """ .

5. Inline SPARQL Query

The sparql attribute points to the full text of a SPARQL.

<sol-query
  endpoint="http://example.com/foo.ttl"
  sparql="SELECT ?s WHERE { ?s ?p ?o . }"
></sol-query>

6. Replaceable Parameters in SPARQL Queries

<sol-query
  endpoint="http://example.com/foo.ttl"
  sparql="SELECT ?s WHERE { ?s ?p ?o } LIMIT {{limit}}"
  var-limit="5"
></sol-query>

Replaceable Parameter syntax:

  • In query: {{paramaterName}}
  • In attribute: var-parameterName="value"

Multiple named parameters can be used in a single query.
