sol-tabs Component

A light-DOM tabbed panel. Fill with <a href="…">Label</a> children — each anchor becomes a tab whose content is loaded from the href. A per-anchor handler attribute picks which sol-* component wraps the URL (falling back to sol-include).

Attributes

AttributeDescription
orientationhorizontal (default) or vertical — stacks the tab bar on the left.
handlerDefault component tag for all tabs. Overridden by a per-anchor handler.

Per-anchor: handler="sol-*" chooses the component to render the href. Any other attributes on the anchor (e.g. wanted, sparql, view) are forwarded to the generated element. The href is passed as both source and endpoint.

Event: sol-tabs-change fires on switch, detail: { name }.

Standalone demo

Three tabs — a Markdown file via sol-include, a Turtle file shown raw, and the same Turtle rendered as a SPARQL table via sol-query.

<sol-tabs>
  <a href="../data/sample.md">Readme</a>
  <a href="../data/sample-data.ttl" raw>Turtle source</a>
  <a href="../data/sample-data.ttl" handler="sol-query" wanted="? ? ?">Triples table</a>
</sol-tabs>

Per-tab handlers

Any sol-* component may be named. The anchor's remaining attributes are passed through, so a tab can drive a sol-query with its own sparql/view, a sol-live-edit with format hints, etc.

<a href="data.ttl" handler="sol-query" view="pie" sparql="…">Chart</a>
<a href="notes.md" handler="sol-live-edit">Editable</a>

As a sol-query view

<sol-query view="tabs"> turns a result set into tabs. The view uses ?link + ?label vars when present, otherwise the first two columns (1st = link, 2nd = label). A handler attribute on the host is forwarded to the generated <sol-tabs>.

<sol-query
  view="tabs"
  handler="sol-include"
  endpoint="../data/sample-data.ttl"
  sparql="
    PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
    SELECT ?link ?label WHERE {
      ?link a ?t ; rdfs:label ?label .
    } LIMIT 4
  "></sol-query>
