sol-include Component

Fetches a remote resource and displays its content inline. Supports HTML, Markdown, and plain text. An optional CSS selector filters to a section of the document. Content is sanitized with DOMPurify by default.

Attributes

AttributeTypeDescription
sourcestring (required) URL of the resource to fetch. Content type is detected from Content-Type header or file extension.
wantedstring (optional) CSS selector. Only elements matching the selector are shown. Works on HTML and rendered Markdown.
rawboolean flag Display source text verbatim in a <pre> block without parsing or sanitizing.
trustedboolean flag Skip DOMPurify sanitization. Only use for content you fully control.
Content is always sanitized by DOMPurify unless trusted is set. The raw attribute bypasses rendering but sets content via textContent, which is safe regardless.

HTML pages

Fetch and render an HTML page inline. Content is sanitized before insertion.

<sol-include source="https://example.org/article.html"></sol-include>

Markdown files

Files with .md / .markdown extension (or text/markdown content-type) are converted to HTML via marked then sanitized and rendered.

<sol-include source="README.md"></sol-include>
Requires marked to be available — either via importmap or window.marked. See CDN setup.

CSS selector (wanted)

The wanted attribute is a CSS selector applied after the content is fetched and parsed. Only matching elements are rendered.

<!-- Only the first <article> from an HTML page -->
<sol-include
  source="https://example.org/blog.html"
  wanted="article:first-of-type"
></sol-include>

<!-- Only h2 headings from a Markdown doc -->
<sol-include
  source="docs/guide.md"
  wanted="h2"
></sol-include>

<!-- All links in a nav section -->
<sol-include
  source="index.html"
  wanted="nav a"
></sol-include>

Raw display

The raw attribute shows the source as plain text in a <pre> block. Useful for displaying code files or the raw source of Markdown/HTML.

<sol-include source="script.js" raw></sol-include>
<sol-include source="README.md" raw></sol-include>

Trusted content

Set trusted to skip DOMPurify sanitization. Only use this for content you fully control (e.g., your own server). Never use trusted for user-generated or third-party content.

<sol-include source="/internal/widget.html" trusted></sol-include>
With trusted set, scripts and event handlers in the fetched content will execute. This is a deliberate choice for trusted single-origin content only.

CDN setup

Add to your HTML <head>:

<!-- Importmap: rdflib (shared), dompurify, marked -->
<script type="importmap">{"imports":{
  "rdflib":    "https://esm.sh/rdflib",
  "dompurify": "https://esm.sh/dompurify",
  "marked":    "https://esm.sh/marked"
}}</script>

<!-- Expose globals -->
<script type="module">
  import * as $rdf      from 'rdflib';
  import DOMPurify      from 'dompurify';
  import { marked }     from 'marked';
  window.$rdf      = $rdf;
  window.DOMPurify = DOMPurify;
  window.marked    = marked;
</script>

<!-- Component -->
<script type="module" src="./sol-include.js"></script>
Only dompurify and marked are needed for sol-include. rdflib is only required if you are also using sol-query on the same page.
