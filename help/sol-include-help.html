sol-include component

Fetches a document and displays its content inline.
Supports authenticated Solid fetch.
An optional CSS/RDFa selector filters documents before rendering.
Content is sanitized with DOMPurify by default.

Required attribute : source Optional attributes : selector raw trusted
See Also : including components using in node using in a browser

Attributes

source attribute

The source attribute is the URL of a text-based file to fetch and render inline. If the source is markdown, it will be converted to HTML.

Fetch and render an HTML page inline. 
<sol-include source="https://example.org/article.html"></sol-include>

selector attribute

The selector attribute is a CSS/RDFa selector applied as a filter to the content.

Fetch an HTML page but only render the first <article>. 
<sol-include
  source="https://example.org/blog.html"
  selector="article:first-of-type"
></sol-include>
Fetch an HTML page and filter by an RDFa attribute. 
<sol-include
    source="https://solidproject.org/TR/index.html"
   selector='a[typeof="doap:Specification"]'
></sol-include>

raw attribute

The raw attribute shows the source as plain text in a <pre> block. Useful for displaying code files or the raw source of Markdown/HTML. 
<sol-include source="script.js" raw></sol-include>

trusted attribute

The trusted attribute disables the DOMPurify sanitizer. An HTML page containing javascript included without this attribute will treat the javascript a plain text. With this attribute, the javasript will execute on load. Use only with trusted sources. 
<sol-include source="script-containing.html" trusted></sol-include>

Attributes

AttributeTypeDescription
sourcestring (required) URL of the resource to fetch. Content type is detected from Content-Type header or file extension.
selectorstring (optional) CSS selector. Only elements matching the selector are shown. Works on HTML and rendered Markdown.
rawboolean flag Display source text verbatim in a <pre> block sanitized but not parsed.
trustedboolean flag Skip DOMPurify sanitization. Only use for content you fully control.
Content is always sanitized by DOMPurify unless trusted is set. The raw attribute bypasses DOMPurify but sets content via textContent, which sanitizes it.

HTML pages

Fetch and render an HTML page inline. Content is sanitized before insertion.

<sol-include source="https://example.org/article.html"></sol-include>

Markdown files

Files with .md / .markdown extension (or text/markdown content-type) are converted to HTML via marked then sanitized and rendered.

<sol-include source="README.md"></sol-include>
Requires marked to be available — either via importmap or window.marked.

CSS selector (pattern)

The selector attribute is a CSS selector applied after the content is fetched and parsed. Only matching elements are rendered.

<!-- Only the first <article> from an HTML page -->
<sol-include
  source="https://example.org/blog.html"
  selector="article:first-of-type"
></sol-include>

<!-- Only h3 headings from a Markdown doc -->
<sol-include
  source="docs/guide.md"
  selector="h3"
></sol-include>

<!-- All links in a nav section -->
<sol-include
  source="index.html"
  selector="nav a"
></sol-include>

Raw display

The raw attribute shows the source as plain text in a <pre> block. Useful for displaying code files or the raw source of Markdown/HTML.

<sol-include source="script.js" raw></sol-include>
<sol-include source="README.md" raw></sol-include>

Trusted content

Set trusted to skip DOMPurify sanitization. Only use this for content you fully control (e.g., your own server). Never use trusted for user-generated or third-party content.

<sol-include source="/internal/widget.html" trusted></sol-include>
With trusted set, scripts and event handlers in the fetched content will execute. This is a deliberate choice for trusted single-origin content only.
