sol-pod : browse and manage Solid pods

  • Full-featured single-panel pod file browser as a web component
  • Browse containers, view/edit files, manage permissions (WAC ACL)
  • Drag-and-drop, rename, delete, create files and folders
  • Live editing for Turtle, JSON-LD, CSV, Markdown, Mermaid, HTML, Graphviz
  • Auto-discovers pods from the current origin, or use the source attribute
  • Pairs with <sol-login> for authenticated access

Full API reference: SolPod API docs · SolPodOps API docs (generated from source JSDoc)

Usage

Basic With Login Two Panels Events Playground Script API → sol-login → sol-modal →

1. Browse a Public Pod

Point source at any publicly readable pod storage root.

<sol-pod source="https://jeff-zucker.solidcommunity.net/"></sol-pod>

2. With Authentication

Pair with <sol-login> for authenticated access. The login attribute is a CSS selector.

<sol-login id="auth" issuers="https://solidcommunity.net"></sol-login>
<sol-pod source="https://myserver.example/" login="#auth"></sol-pod>

3. Two-Panel Layout

Use two <sol-pod> elements side by side. Drag files between panels to copy.

<sol-login id="auth"></sol-login>
<div style="display:flex; gap:8px; height:500px">
  <sol-pod source="https://pod-a.example/" login="#auth" style="flex:1"></sol-pod>
  <sol-pod source="https://pod-b.example/" login="#auth" style="flex:1"></sol-pod>
</div>

4. Listening to Events

<sol-pod id="browser" source="https://pod.example/"></sol-pod>
<script>
  document.getElementById('browser').addEventListener('sol-navigate', (e) => {
    console.log('Navigated to:', e.detail.url);
  });
  document.getElementById('browser').addEventListener('sol-status', (e) => {
    console.log('[' + (e.detail.type || 'info') + ']', e.detail.message);
  });
</script>
